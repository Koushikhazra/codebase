# Project structure

This file documents the recommended project structure for CodeBase (DSA Revision Hub).

Root
├─ .env.example           # example environment variables (do NOT commit real secrets)
├─ .gitignore
├─ README.md
├─ package.json
├─ vite.config.js
├─ postcss.config.js
├─ tailwind.config.js
├─ server/                # Express backend
│  ├─ server.js           # Entry point (connects DB, mounts routes)
│  ├─ middleware/
│  │  └─ auth.js         # JWT auth middleware
│  ├─ models/
│  │  ├─ User.js
│  │  ├─ Question.js
│  │  └─ Topic.js
│  └─ routes/
│     ├─ auth.js
│     ├─ questions.js
│     └─ topics.js
├─ src/                   # Frontend (Vite + React)
│  ├─ main.jsx
│  ├─ index.css
│  ├─ App.jsx
│  ├─ components/
│  │  ├─ HomePage.jsx
│  │  ├─ QuestionsPage.jsx
│  │  ├─ RevisionPage.jsx
│  │  └─ LoginForm.jsx
│  ├─ context/
│  │  └─ AppContext.jsx
│  └─ services/
│     └─ api.js           # Frontend API client (use VITE_API_BASE_URL)
├─ dist/                  # Generated by `npm run build` (ignored in git)
└─ .github/               # Optional CI/CD (workflows)

Notes
- `VITE_API_BASE_URL` environment variable controls the frontend API base (baked at build time).
- For single-service deployment (Render), have the backend serve `dist/` and set `VITE_API_BASE_URL=/api`.
- Keep secrets out of source control; commit `.env.example` but _not_ `.env`.
- Add a `start` script in `package.json` for production: `node server/server.js`.

Suggested additions
- `.env.example` (this repo includes one)
- `.github/workflows/ci.yml` to run lint/tests on PRs
- `Dockerfile` (optional) for containerized deploys
- `CONTRIBUTING.md` and `LICENSE`

This is a living document — update as the project grows.
